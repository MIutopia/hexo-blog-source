## 前言

上一篇我们讲了 Git 回滚的五种方法——`checkout`、`reset`、`revert`、`stash`、重新克隆。如果你认真学了，应该能应对大部分翻车场景。

但说实话，对很多新手来说，记住这些命令的区别还是挺头疼的。什么时候用 `--soft`，什么时候用 `--hard`？`revert` 和 `reset` 到底该选哪个？

**如果不用记这些命令呢？**

如果你能直接用中文告诉 Claude Code："把刚才那次改动撤掉"、"回到十分钟前的状态"、"先存个档再继续改"——它自己去执行对应的 Git 命令，你只需要看结果对不对。

这篇文章就讲怎么做到这件事。

## 核心思路：让 Claude 当你的 Git 翻译官

Git 命令是给人看的操作界面，但本质上它做的事情很简单：保存快照、对比差异、恢复状态。这些逻辑 Claude Code 完全理解。

你要做的只有两件事：

1. **在 CLAUDE.md 里定好规则**，让 Claude 在干活时自动做好安全防护
2. **用自然语言指挥回滚**，不需要记任何 Git 命令

## 第一步：在 CLAUDE.md 里写好"安全规范"

打开你项目根目录的 `CLAUDE.md`，加上这样一段规则：

```markdown
## 安全规范

### 操作前自动存档
- 每次修改文件前，先 git add + git commit 当前状态，提交信息格式：
  "checkpoint: 操作前的安全存档 - [简要描述即将执行的操作]"
- 这样每次改动都有一个可以回退的节点

### 批量操作前额外保护
- 批量修改超过 3 个文件前，先创建一个新的 Git 分支
- 在分支上完成操作，确认无误后再合并回 main

### 回滚响应
- 当我说"撤回"、"回滚"、"撤销"、"回到之前"时，
  自动执行 git revert 或 git reset（根据是否已推送判断）
- 当我说"存档"时，执行 git stash
- 当我说"恢复存档"时，执行 git stash pop
```

有了这段规则，Claude Code 每次启动都会读到它，然后按照这个规范来工作。

## 实际效果：用自然语言完成整个流程

下面模拟一个完整的工作场景，看看实际用起来是什么样的。

```
你：帮我把 source/_posts/ 下所有旧文章统一改写成最新那篇的风格。

Claude：好的，我先存档当前状态，然后开始逐篇改写。
       [自动执行 git commit -m "checkpoint: 操作前的安全存档 - 批量改写文章格式"]
       开始改写第 1 篇...
```

注意这里 Claude **自动先做了存档**，不需要你提醒。

```
你：等一下，第 2 篇改得不对，把第 2 篇恢复成原来的。

Claude：好的，我把 article2.md 恢复到上次存档的状态。
       [自动执行 git checkout HEAD~1 -- source/_posts/article2.md]
       已恢复，你看一下对不对。
```

你不需要知道 `git checkout HEAD~1 --` 这个命令，只需要说"恢复成原来的"。

```
你：其他文章改得没问题，继续改剩下的。

Claude：好的，继续改写第 3 篇到第 15 篇...
       [改完后自动 commit]
       全部改完了，要我直接推送到 GitHub 吗？
```

```
你：先别推，我本地看看效果。

Claude：好的，暂不推送。你可以在本地预览，有问题随时告诉我。
```

```
你：看了下效果不太好，把刚才的批量改动全部撤掉。

Claude：好的，我撤销最近那次批量改写的 commit。
       [自动执行 git revert HEAD 或 git reset --soft HEAD~1]
       已撤销，所有文章恢复到改写前的状态。
```

整个过程你没有输入过一条 Git 命令。

## 常用的自然语言回滚指令

整理一些你在实际工作中会用到的自然语言指令，直接对 Claude Code 说就行：

| 你说的话 | Claude 会做的事 |
|---------|---------------|
| "撤回上一步" | `git reset --soft HEAD~1`（还没 push）或 `git revert HEAD`（已 push） |
| "把刚才改的 XX 文件恢复原样" | `git checkout HEAD~1 -- 文件名` |
| "全部撤掉，回到改之前" | `git reset --hard HEAD~N`（会先跟你确认） |
| "先存个档" | `git stash` |
| "恢复存档" | `git stash pop` |
| "看看之前改了什么" | `git diff` 或 `git log --oneline` |
| "回到昨天下午的状态" | 根据 `git log` 的时间戳找到对应 commit，执行 reset 或 revert |
| "这个改动太危险，开个新分支做" | `git checkout -b 新分支名` |

你不需要记住这些命令，**只需要用你习惯的方式描述你想做什么**。

## 进阶：让 Claude 自己判断风险

在 CLAUDE.md 里可以加更智能的规则，让 Claude 自己判断哪些操作需要额外保护：

```markdown
## 风险判断规则

### 高风险操作（必须自动存档 + 通知用户）
- 删除文件
- 批量修改超过 5 个文件
- 修改配置文件（_config.yml、vercel.json 等）
- 修改主题文件

### 中风险操作（自动存档，静默执行）
- 修改单篇文章内容
- 添加新文件
- 修改 frontmatter

### 低风险操作（直接执行）
- 读取文件
- 查看 git 状态
- 查看部署日志
```

这样 Claude 在删除文件之前会自动先 commit 存档，你不用担心它"静默删库"。

```
Claude：好的，已经推送到远程了，我用 git revert 来撤销。
       [git revert HEAD --no-edit]

       已撤销并推送，所有文章的 description 字段已移除，
```

## 对比：传统方式 vs 自然语言方式

| 操作 | 传统方式（你需要记的命令） | 自然语言方式 |
|------|------------------------|------------|
| 撤销未提交的改动 | `git checkout -- 文件名` | "把 XX 恢复原样" |
| 撤回一次 commit | `git reset --soft HEAD~1` | "撤回上一步" |
| 撤销已推送的改动 | `git revert HEAD` + `git push` | "把刚才推送的撤掉" |
| 临时保存现场 | `git stash` | "先存个档" |
| 查看改了什么 | `git diff` / `git log` | "看看之前改了什么" |
| 回到某个时间点 | `git log` → 找 hash → `git reset` | "回到昨天下午的状态" |

传统方式你需要记住命令、理解参数、判断用哪个。自然语言方式你只需要**描述你想达到的效果**。

## 注意事项

虽然自然语言回滚很方便，但有几点还是要了解：

**Claude 会先确认再执行危险操作。** 如果你在 CLAUDE.md 里写了"高风险操作前通知用户"，Claude 在执行 `git reset --hard` 这种不可逆操作前会先问你确认。这是好事——给 AI 加一道人工审核。

**推送后的回滚比推送前复杂。** 没推送的改动，Claude 可以直接 reset；已经推送的，需要用 revert 生成新 commit。你不需要理解这个区别，但知道"推出去的东西撤回来会多一步"有助于你养成"改完先看再推"的习惯。

**频繁存档会产生很多 commit。** 如果你介意 git 历史太乱，可以让 Claude 在任务完成后把多个 checkpoint commit 合并成一个（用 `git rebase -i`，但这个操作比较复杂，建议让 Claude 自己处理）。
