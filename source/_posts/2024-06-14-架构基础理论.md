## 前言

在软件开发和系统设计中，"架构"是一个核心概念。这篇文章简要介绍前端、后端、接口等基础概念。

## 架构概述

### 前端

前端是用户看得见的页面（小程序、Web 网页、App），负责将后端处理好的数据进行处理和展示。

- **静态页面**：页面内容被写死，不会动态变化
- **开发语言**：Java、HTML、CSS、JavaScript 等

前端代码可以存放于任何有网且能够连接到后端数据库的地方，运行于客户端（如浏览器、App）。

### 后端

后端负责处理业务逻辑、数据存储和计算。它接收前端的请求，进行处理后返回结果。

- **开发语言**：Java、Python、Go、Node.js 等
- **数据库**：MySQL、PostgreSQL、MongoDB 等

### 接口

接口是前后端通信的桥梁，定义数据交换的格式和规范。

常见的接口规范：

| 规范 | 说明 |
|------|------|
| RESTful | 基于 HTTP 方法的接口风格 |
| GraphQL | 灵活的数据查询语言 |
| gRPC | 高性能远程过程调用框架 |

## 典型架构

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   前端      │────→│   接口      │────→│   后端      │
│  (浏览器)   │←────│  (API)      │←────│  (服务器)   │
└─────────────┘     └─────────────┘     └─────────────┘
                                              │
                                              ↓
                                        ┌─────────────┐
                                        │   数据库    │
                                        └─────────────┘
```

## 高可用架构

为了保证服务的可用性，通常会采用集群部署：

- **Nginx + Keepalived**：实现 Web 服务的高可用
- **数据库主从复制**：实现数据的高可用
- **Redis 集群**：实现缓存的高可用
